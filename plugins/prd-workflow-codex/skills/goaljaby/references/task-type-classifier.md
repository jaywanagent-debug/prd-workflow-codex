# 작업 유형 분류 규칙 (Step 1)

PRD 본문 텍스트에서 작업 유형 6종 중 하나를 추정한다. Step 3 인터뷰(§A 번호 블록)에서 사용자가 최종 확정한다 — 자동 추정값을 추천 1순위 옵션(1번)으로 제시.

매칭 기준은 **한·영 키워드 동시 매칭**. 영어 키워드만 보면 한글 PRD가 항상 "기능 구현"으로 잘못 떨어진다.

## 매칭 표

각 키워드 출현 횟수 × 가중치를 합산해 유형별 점수를 낸다.

| 유형 | 한국어 키워드 | 영어 키워드 | 가중치 |
|------|--------------|-------------|--------|
| UI 구현 | "UI", "화면", "디자인", "레이아웃", "viewport", "스크린샷", "버튼", "폼", "모달", "반응형" | "UI", "viewport", "screenshot", "layout", "responsive", "design system" | 키워드당 +2 |
| 버그 수정 | "버그", "오류", "수정", "재현", "회귀", "이슈", "픽스" | "bug", "fix", "regression", "reproduce", "repro", "issue" | 키워드당 +2 |
| 마이그레이션 | "마이그레이션", "이전", "이관", "패리티", "롤백", "호환성", "교체" | "migration", "migrate", "parity", "rollback", "legacy" | 키워드당 +3 |
| eval 개선 | "프롬프트", "평가", "evaluation", "점수", "정확도", "벤치마크", "베이스라인", "튜닝" | "prompt", "eval", "score", "accuracy", "baseline", "benchmark" | 키워드당 +3 |
| 문서 집필 | "문서", "가이드", "리포트", "블로그", "섹션", "톤", "글", "집필", "초안", "원고" | "document", "guide", "report", "blog", "section", "tone", "draft" | 키워드당 +2 |
| 기능 구현 | "기능", "추가", "구현", "개발", "신규", "API", "엔드포인트" | "feature", "implement", "add", "new", "endpoint" | 키워드당 +1 (기본값) |

## 매칭 절차

1. PRD 디렉토리의 모든 `.md` 파일 본문을 합친다
2. 본문을 lowercase로 정규화하고 영어 키워드 검색
3. 한국어 키워드는 원문 그대로 검색
4. 각 키워드 출현 횟수 × 가중치 합산
5. 최고점 유형 선택
6. **동점 시 우선순위**: 마이그레이션 > eval 개선 > UI 구현 > 버그 수정 > 문서 집필 > 기능 구현 (특수→일반 순)
7. 어떤 유형도 0점 초과를 못 받으면 → **기능 구현** (fallback)

Bash로 결정론적 구현 예시:

```bash
# 모든 PRD .md 본문 합치기
CONTENT=$(cat {PRD_DIR}/*.md)

# 한 유형 점수 계산 예 (UI 구현)
UI_KO=$(echo "$CONTENT" | grep -oE "UI|화면|디자인|레이아웃|스크린샷|버튼|폼|모달|반응형" | wc -l)
UI_EN=$(echo "$CONTENT" | grep -oiE "viewport|screenshot|layout|responsive|design system" | wc -l)
UI_SCORE=$(( (UI_KO + UI_EN) * 2 ))
```

(나머지 5개 유형도 같은 방식. 합산 후 최고점 선택.)

## Step 3 표시 형식 (§A 번호 블록)

자동 추정 결과를 §A 번호 블록으로 사용자에게 보여준다. 추정값을 항상 1번(추천)에 둔다:

```text
작업 유형 자동 추정: UI 구현 (점수 14, 매칭 키워드: 화면×3, 스크린샷×2, viewport×1, 버튼×4, layout×1)

질문: 작업 유형이 맞나요?
1. UI 구현 (추천)
2. 기능 구현
3. 버그 수정
4. 문서 집필
5. 마이그레이션
6. eval 개선
7. 문장으로 직접 수정 요청
```

추정 점수가 1~3점으로 약하면 "(추천)" 표기를 빼고 유형 6개를 평평하게 나열한다 — 낮은 신뢰도 신호. PRD에서 작업 유형이 압도적으로 명백하면 §2c에 따라 질문을 생략하고 기본값으로 확인만 한다.
